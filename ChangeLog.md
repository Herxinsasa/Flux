## 一、UI / UE（界面与交互）

| 主题 | 问题 / 诉求 | 修改方案概要 |
|------|----------------|--------------|
| 设置页气泡位置 | 成功/失败提示希望在上方 | `SettingsToast` 使用 `position: fixed; top: 72px`，后用 **`createPortal(..., document.body)`** 避免被父级遮挡 |
| 浅色主题设置页对比度 | AI 设置区描边浅、部分无描边 | 滚动区增加 **`flux-settings-surface`**；`theme.css` 中浅色 **`--settings-stroke`**；API Key / Base URL 等容器统一 **`1px solid var(--settings-stroke, …)`** |
| 设置页文案 | 不需要展示配置文件路径 | 移除底部路径说明块；Toast 仅保留「保存成功 / 失败」「连接测试成功 / 失败」等短文案 |
| Skill 管理页 | 删除冗长「提示语」 | `SkillPanel` 移除顶栏下关于 userData/skills 路径的长说明段落 |
| AI 对话输入 | Enter 发送、占位文案 | **`ChatInput`**：默认 **Enter 发送**，**Shift+Enter** 换行；占位 **「请输入问题...」** |
| AI 对话滚动条 | 滚动条适配主题 | **`theme.css`**：对话区 Virtuoso 滚动条使用 **`scrollbar-color` / webkit-thumb** 与主题变量 |
| AI 对话布局 | 常规 IM：己方右、对方左 | **`MessageLine` + theme**：**`.msg-row--user` 右对齐**、**`.msg-row--ai` 左对齐**；气泡 **`msg-bubble--user` / `--ai`** |
| AI 回复展示 | Markdown 应用预览而非源码 | AI 侧使用 **`MdPreview`**；**`hideEmptyPlaceholder`** 避免空气泡出现编辑器占位文案 |
| 对话跟帖 | 回复后可见最新内容 | **`Virtuoso`**：`followOutput`、`alignToBottom`、`increaseViewportBy`；**`useEffect`** 在消息变化时 **`scrollToIndex` 至末尾** |
| 工作区文件树 | 打开文件夹后目录全展开，信息过载 | **`WorkspaceTreePanel`**：去掉「挂载时收集全部目录 key 并展开」的逻辑；**默认全部折叠**，仅切换/清空工作区时重置展开状态 |

## 二、配置 / 存储 / 路径

| 主题 | 问题 / 诉求 | 修改方案概要 |
|------|----------------|--------------|
| 配置文件位置 | `flux-settings.json` 落在工作路径、`config` 目录 | **`src/main/paths.ts`**：`getWorkRoot()`（打包 = exe 目录，开发 = **`process.cwd()`**）→ **`getConfigDir()`** = `{workRoot}/config`，自动 **`mkdir`**；**electron-store** 使用 **`cwd: getConfigDir()`**，文件名 **`flux-settings`** |
| Skill 用户文件位置 | 与配置同级策略，便于携带 | **`getUserSkillsRoot()`**、`getUserSkillPackagesDir()`：工作目录下 **`skills/`**、**`skills/packages/`**；不可写时回退 **`userData`** |
| 设置 IPC | 不在界面暴露路径 | 移除 **`configStoragePath`** 的 IPC / Zustand 字段 |

## 三、设置模块（Settings）BUG / 逻辑

| 主题 | 问题 / 诉求 | 修改方案概要 |
|------|----------------|--------------|
| 新建提供商 + 更新丢数据 | 点击测试/保存后提供商列表被清空 | **`updateProvider` / `deleteProvider`** 改为基于 **`useSettingsStore.getState().providers`** 更新，避免 **`addProvider` 同一次点击闭包仍是旧数组**导致 `setProviders([])` |
| 测试连接无反馈 | 点击测试无 Toast | 修复上一条后保存恢复正常；Toast **`Portal`** + **`z-index`**；**`handleTest`** 外层 **`try/catch`** |
| 测试连接用错 Key | save()+load() 后列表中的 Key 为脱敏串 | 测试时使用 **表单内明文 `apiKey`** 调用 **`testConnection`**；若表单含 **`***`** 则提示重新输入完整 Key |
| 切换预设与表单 | 切换预设后配置像被重置 | 切换预设时 **`setActiveProvider(null)`** 并 **`save()`**；用 **`prevActiveIdRef`** 区分「取消选中已保存项」与「仅换预设」，避免误清空正在输入的 Key |
| 重新进入设置页 | 预设类型变回 Anthropic、与已保存不符 | 增加 **`inferPresetKeyFromProvider()`**；选中 **`activeProvider` 时同步 `presetKey`** |
| 连接失败提示过曝 | 弹窗直接显示服务端 400/401 细节，用户噪声高 | 设置页与全局自动校验失败提示统一为 **「连接失败，请检查配置」**；详细错误写入主进程日志 |
| 保存配置必填校验缺失 | 可保存空 API Key/空模型，导致后续连通性不稳定 | 前端保存前新增必填拦截（API Key/模型/自定义 Base URL）；主进程 `SETTINGS_SAVE` 增加兜底校验，防止 IPC 绕过 |
| 模型清单含临时命名 | `[1m]`/日期快照等命名导致兼容风险与认知负担 | 供应商模型列表收敛为主流稳定别名；DeepSeek 默认回到 `deepseek-v4-flash`，移除临时/快照项 |

## 四、聊天 / Agent 模块

| 主题 | 问题 / 诉求 | 修改方案概要 |
|------|----------------|--------------|
| 搜索面板冲突 | CodeMirror 搜索面板与自定义面板同时出现 | **`useEditor`** 里使用 `@codemirror/search` 官方能力并隐藏原生 panel DOM，仅保留自定义 `SearchPanel` |
| Ctrl+F 再打开异常 | 搜索关闭后无法稳定再次打开 | 移除冲突的 `searchKeymap` 接管，统一由外层快捷键触发搜索面板开关 |
| `.log` 文件体验不一致 | 字体、行为与普通文本编辑器不一致 | `.log` 路由统一走 `EditorPane`；字体、行高、滚动表现与 `.txt/.py/.cpp` 对齐 |
| 日志语义识别弱 | 日志级别不易快速扫描 | 新增日志级别着色（INFO/DEBUG/WARN/ERROR/FATAL）扩展，保持编辑器统一但可视层次更清晰 |
| 编辑器缩放体验 | 需要类 IDE 的快捷缩放 | 增加 `Ctrl/Cmd +`、`Ctrl/Cmd -`、`Ctrl/Cmd 0` 字号缩放；去除滚轮缩放监听，避免滚动卡顿 |
| Agent 过程可见性 | 用户难感知工具执行进度 | 主进程增加 tool progress 事件与日志，渲染侧展示阶段提示并在异常/取消时做清理 |
| 工具结果可读性 | `read_file/write_file` 卡片信息过载 | 卡片改为“摘要优先 + 默认折叠详情”，并保持展开查看原始输入/输出能力 |
| 对话-编辑器联动 | AI 写入后定位不直观 | 写入成功后通过 IPC 回传修改行区间，编辑器自动高亮改动范围并滚动到可视区域 |
| 输入区交互体验 | 输入区不够整体、发送操作不聚焦 | 发送按钮嵌入输入框（上箭头样式），输入区改为整体卡片，去除冗余提示文案，placeholder 改为“输入你的想法...” |
| AI 气泡间距 | 回复气泡顶部留白偏大 | 收紧列表项间距与 AI 气泡内边距，提升连续阅读密度 |
| 快捷键引用选区 | Alt+K 只能引用一次 | 引用改为“追加模式”：最多保留 5 段、按内容去重；输入区显示多引用 chip，发送时按段拼入 `preface` |
| 底部状态栏简化 | “就绪”与行列信息噪声偏高 | 保留状态点和文件信息，移除“就绪/运行中”文本与“第 X 行第 Y 列”显示 |
| `@` 文件上下文 | 工作区索引 + 已打开文件可选 | **`mentionFiles`** = **`workspaceFiles` ∪ `editorFiles`** 去重、按名称排序；发送前 **`file.read`** 拼入 **`preface`**（大文件截断） |
| 对话历史角色 | 主进程需要正确的 assistant / user | **`buildSendContext`** 中 history：**`user` / `assistant`**，assistant 带 **`reasoningContent`** |
| Skill 可用性说明 | 调用已安装的 Skill 包 | **`buildSystemPrompt`** 中 **`SkillManager.list()`** 注入已启用 Skill **名称列表**；**`buildSkillSystemPrompt`** 仍按关键词注入 Skill **正文** |
| DeepSeek `reasoning_content` 400 | 思考模式多轮须回传推理文本 | **流式**：解析 **`reasoning_delta`** → 存入消息的 **`reasoningContent`**；**`toOpenAiMessages`**：**合并连续多条 assistant+tool 为单条 `tool_calls`**，并带上 **`reasoning_content`**；**工具参数**用 **`serializeToolCallArguments`（`input` JSON）**，禁止空 **`content` 当 arguments** |

## 五、Markdown 编辑器（实时预览 / 切换文件 / 保存）

| 主题 | 问题 / 诉求 | 修改方案概要 |
|------|----------------|--------------|
| 预览区标题样式 | MDX 预览与 Typora 差距大，h1/h2 无法区分、无加粗放大感 | **Tailwind Preflight** 将 **`h1–h6`** 设为 **`font-size/font-weight: inherit`**；在 **`theme.css`** 中为 **`.flux-mdx-editor-root .flux-mdx-content`** 下的 **h1–h6** 补充分级 **字号、字重、间距**，并为 **`strong`** 加重 |
| 切换已打开文件 | 点击另一 tab 时短暂显示上一个文件内容 | **`fileStore.loadFileContent`** 开始时 **`useEditorStore.setState({ content: '', isDirty: false })`** 立即清空；**`read` 完成**与 **流式 chunk / 结束** 处均判断 **`currentFile === filePath`** 再写入，避免异步晚到覆盖当前文件 |
| Ctrl+S 保存 | 在预览/可编辑区域保存不生效 | **`useShortcuts`**：`keydown` 使用 **捕获阶段**（**`addEventListener(..., true)`**）；Ctrl+S 上 **`preventDefault` + `stopPropagation`**；成功后 **`markClean()`** |
