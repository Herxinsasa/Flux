# Flux 使用帮助

欢迎使用 Flux。这个页面覆盖最常用的工作流，帮助你在 3-5 分钟内上手。

## 1. 快速开始

1. 打开文件或文件夹。
2. 在右侧聊天区输入问题，或使用 @ 文件把上下文带给 Agent。
3. 需要特定能力时，使用 /Skill 显式调用，或在 Skill 管理页维护技能。
4. 需要导出分析结果时，使用导出功能生成 Markdown 报告。

## 2. 主界面说明

### 左侧

- 文件树：浏览和选择工作区文件。
- 工作集/文件列表：每个已打开文件右侧有 `×`，可快速关闭该文件。
- 顶部图标：
  - 闪电：Skill 管理。
  - 太阳：主题切换。
  - 齿轮：设置。

- 顶部菜单：
  - 文件 / 编辑 / 段落 / 查看 / 帮助 这一排标准叫“菜单栏”，日常也可叫“功能栏”。
  - 编辑菜单支持剪切、复制、粘贴、查找和全选；在编辑器和文本输入框中都可使用。
  - 段落菜单可快速插入常用结构：一级标题、二级标题、水平分割线；适合快速整理文档层级。
  - 使用段落菜单插入内容时，会在当前光标位置写入，不会覆盖未选中的文本。
  - 若当前有选中文本，段落菜单会替换选中区域，便于快速重构章节内容。
  - 建议先插入标题再补正文，能更快形成清晰的大纲结构。
  - 帮助 > 使用说明：打开帮助中心。
  - 帮助 > 关于：查看应用图标、当前版本和项目主页链接。

### 中间

- 编辑器区：查看和编辑文件。
- Markdown 文件支持预览和大纲联动。

### 右侧

- AI 对话区：进行日志/代码分析。
- 支持工具调用流式反馈、进度提示和结果回显。

## 3. Skill 使用

### 导入 Skill

- 支持导入单文件（.md）或目录（技能包）。
- 目录导入适合包含模板、脚本、附加说明的 Skill。

### 卸载 Skill

- 右上角删除按钮是软件内卸载，不会删除磁盘文件。
- 内置 Skill 不支持删除。

### 失效 Skill

- 当入口文件或资源目录缺失时，Skill 会显示已失效。
- 聊天发送前若命中失效 Skill，会给出明确提示并跳过注入。

## 4. AI 对话技巧

### @ 文件引用

- 输入 @ 可选择文件，发送时会把文件内容拼接为上下文。
- 建议优先引用关键片段，避免一次带入过多无关内容。

### /Skill 显式调用

- 输入 / 可选择已安装 Skill。
- 适合强约束输出格式或固定分析流程。

### 引用编辑器选区

- 可把当前选区作为上下文发送，便于局部分析和修复。

## 5. 设置与连接

- 在设置中配置 Provider、模型、API Key。
- 建议先做连接测试，再保存。
- 连接失败时，界面显示通用错误，详细原因写入主进程日志。

## 6. 导出报告

- 分析完成后可导出 Markdown 报告。
- 推荐在导出前检查结论、风险和修复建议是否完整。

## 7. 常见问题

### Q0: 新建文件时默认后缀是什么？

- 默认文件名是 `untitled.md`，与保存对话框首项 Markdown 类型保持一致。

### Q0-1: 新建文件时选择“覆盖已有文件”会发生什么？

- 确认替换后，目标文件会被清空并作为新建文件打开。
- 如果你希望保留原内容，请在保存对话框中改用新文件名。

### Q1: 为什么我导入后看不到 Skill？

- 检查 frontmatter 是否包含 name。
- 检查是否与内置 Skill 重名。
- 重新打开 Skill 管理页刷新列表。

### Q2: 为什么某个 Skill 被标记为失效？

- Skill 的入口文件或资源目录不存在。
- 可在原路径恢复文件，或在 Skill 页卸载后重新导入。

### Q3: 为什么点击外部链接会跳浏览器？

- 这是安全与可用性设计，避免应用内弹窗阻塞主界面。

### Q4: 已打开文件太多，怎么快速整理？

- 在左侧工作集/文件列表中，点击文件行末 `×` 可逐个关闭。
- 建议只保留当前任务相关文件，减少切换成本。

## 8. 建议工作流

1. 打开目标日志或代码文件。
2. 用 1-2 句描述你的目标和约束。
3. 必要时追加 @ 文件和 /Skill。
4. 先拿结论，再让 Agent 生成可执行修复步骤。
5. 导出报告并归档到项目文档。

## 9. 反馈与迭代

如果你希望增加某个快捷能力（例如规则检查、专用报告模板、批处理流程），建议在 Skill 管理页先沉淀为可复用 Skill，再进入常规分析流程。
